What is Hyperion?

Hyperion is a dynamically typed programming language with a garbage collector. Hyperion compiles to bytecode and executes it using a VM (Virtual Machine).


ยง

Variables

In Hyperion, you would use the 'let' keyword to create a variable. 

        let a = 5;

Because the language is dynamically typed, you can do the following. 

        let a = 5;
a = "Hello world!";

Data Types

In Hyperion, there are 4 data types.

  • Numbers
  • Strings
  • Booleans
  • Lists

The critical thing to notice here is that lists can contain different types simultaneously. 

        let a = [
  5,
  "Hello world!",
  true,
  [1, 2, true]
];

When performing operations, the two operands must be the same type. You can't add an integer and a string. Here are all the operators supported in Hyperion.

Operation Numbers Strings Booleans Lists
* 2 * 3 = 6
- 5 - 6 = -1
/ 5 / 2 = 2.5
+ 5 + 5 = 10 "ab" + "oba" = "aboba"
% 10 % 4 = 2
== 5 == 5 "aboba" == "aboba" true == true
>= 5 >= 5, 5 >= 3
<= 5 <= 5, 3 <= 5
> 5 > 3
< 3 < 5
! always false always false !true = false, !false = true always false
^ 10 ^ 3 = 9, basic XOR

While

Hyperion also has while loops. 

        while (<condition>) {
  <body> 
}

ยงยง

For

        for (statement1; statement2; statement3) {
  <body> 
}

Statement 1 is executed once before the execution of the code block.
Statement 2 defines the condition for executing the code block.
Statement 3 is executed every time after the code block has been completed.

Here is an example of how to add 1 to each item in a list in Hyperion. 

              let a = [1, 2, 3];
for (let i = 0; i < 3; inc i) {
  inc a[i];
}
print a;

Functions

To declare a function in Hyperion, use the 'def' keyword. Use the following format. 

        def <function name>(<argument1>, <argument2>, ...) {
  <body>
}

You can use the return keyword to return something from a function. For example, you can write the following to write the function that takes in a number x and returns x * x. 

        def f(x) {
  return x * x;
}

Functions in Hyperion are also values. So you can pass functions as arguments to other functions and store them in variables. 

              def f(x) {
  return x * x;
}
let g = f;
print g(5);

And also, you can declare functions inside of functions. 

              def f(x) {
  def g() {
    print x;
  }
  return g;
}
let x = f(5);
x();

This code will output 5.

Classes

In Hyperion, as in most programming languages, there are classes. To demonstrate how classes work, look at the following example. 

        import std type_conv;

class Person {
  init(age, name) {
    this.age = age;
    this.name = name;
  }

  show() {
    print "[AGE]: " + type_conv:to_string(this.age) + " [NAME]: " + this.name;
  }
}

let Bob = Person(41, "Bob");
Bob.show();

This code will output '[AGE]: 41.000000 [NAME]: Bob'. The first line is an import of a standard library for type conversion.

Modules

You want to split code into multiple files when working on big projects. Let's say you have a file Person.hypl with the following code. 

        import std type_conv;

class Person {
  init(age, name) {
    this.age = age;
    this.name = name;
  }

  show() {
    print "[AGE]: " + type_conv:to_string(this.age) + " [NAME]: " + this.name;
  }
}

And we have a main.hypl file. We want to import Person from Person.hypl to do that, we can write the following. 

        import module Person;

let Bob = Person(41, "Bob");
Bob.show();

And this code will output '[AGE]: 41.000000 [NAME]: Bob'. If you are importing files from other folders you can use the @ symbol for example. If the Person class would be in libraries/person.hypl, you would rewrite the main.hypl the following way. 

        import module libraries@Person;

let Bob = Person(41, "Bob");
Bob.show();
